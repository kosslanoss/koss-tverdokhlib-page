  • Semantic code

    • Markup should be performed using semantic tags

    Heading tags h1 - h6 are used as intended. Following the hierarchy. The h4 heading cannot be placed after the h1 heading

  • BEM metodology

    • Conception BEM is not only a method of naming selectors. It is a paradigm of project perception as a set of entities (blocks, elements, modifiers). A full BEM stack implies double templating and has a relatively high entry threshold.

  • Valid and beautiful code

    • The HTML document must be validated. Use the validator to check.

  • Markup does not break

    • Markup does not break when adding more text to elements, does not break when adding more text to buttons, does not break when using pictures with inappropriate sizes. That is, you need to independently try to add more text, larger images, etc.

  • Dynamic content

    • Content is ready for integration to CMS

  • Inclusivity

    • Interactive elements without text description have description sounded by screenreader

  • Cross-browser markup

    • Markup should be cross-browser displayed in the latest versions of popular browsers: Firefox, Opera, Google Chrome, Safari

  • Responsive

    • Using no more than 4-5 breakpoints. Classic breakpoints are: 1200px, 992px, 768px, 576px

    Interactive elements have a normal clickable zone (at least 44px width and height)

    Content does not stick to the edges of the screen, does not stick to itself, does not have too much space between the block and the edge of the screen

  • Hover/focus/active

    • All interactive elements (links, buttons) must have different styles in :focus, :hover, :active states Fields input, textarea - in state :focus

    Using smoothly animation "transition" in hover effects with the property that we are animating, instead of the "all" value

  • Groupping

    • Classes with same styles are grouped. It helps to no-repeat code

  • Fonts
